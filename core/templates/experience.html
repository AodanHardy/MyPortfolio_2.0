Professional Experience

Intern Software Engineer

Magnite | Belfast | 2023 - 2024

My placement is at a company called Magnite, based at their Belfast office. My placement opportunity began on the 4th of July 2023 and is anticipated to end at the end of June 2024. During my time as an Intern in Magnite, I worked in the Reporting Team, to help build a new Reporting system to replace the Legacy system.

Adapting

When I joined the SpringServe Reporting Team as an intern, I faced a significant learning curve. Initially, I lacked the depth of knowledge required for the role. I found it surprising that despite my experience in Python and C#, the team primarily used Java. However, I had some prior exposure to Java and found it interesting, so I embraced the opportunity to expand my skills in that area. Beyond programming languages, the role demanded various other competencies. Collaborative software development was new to me, having previously worked independently in my last placement. Additionally, understanding the intricate project structure of insight was challenging due to its complexity. Despite these hurdles, I was eager to immerse myself in the learning process and contribute to the team's success.

My Contribution

  • Test Harness
  • Grafana Dashboard
  • Testing

The aim of the test harness is to get the reports done by v1 reporting, and request insight for them, then once insight has done it, get both CSV files and compare them. Once compared, put the results into a Postgres table. My first attempt at this took far too long to do. I kept getting stuck on issues and in the end, it was only able to successfully compare about 10% of the reports. But after a few months of working on different issues I felt much more confident that I was able to make it work a great deal better. I also felt somewhat embarrassed over the quality and structure of the code, since when I was doing it in the first place, I didn’t have a great understanding of Insight and the reporting system, and I wanted to correct this. And in the space of a couple of weeks I had the Test Harness working much better and in a much more structured way and the success rate of 10% increased to over 90%, with the rest of the failures being unavoidable. I was able to do this because of the amount of learning I had done in the months between. This will be vital when comparing the new Reporting system Insight and the old system. The reports generated by insight need to be identical to the old system, and the test harness will tell them how it is progressing in that.

Over my placement I have written many Junit tests. When I started there were only a few JUnit tests written, but by using the site, I was able to see where was covered and where wasn’t, and by writing tests where needed I was able to get the coverage from below 10% to over 65%.

Before I built the Grafana Dashboard, the SpringServe Reporting Team used mostly DataDog to monitor how insight was performing, but most of the data there relates more to CPU and memory usage rather than more relevant data for the team. So, the Grafana dashboard was custom built specially with the relevant data that the team actually want to see, such as the rate of reports, which reporting system is using them, report types, processing times of both systems, fail rates, and the results of the test harness. Everything relating to how Insight is done on one page. After my placement I believe this will still be used daily by the other team members as the primary monitoring tool.

Going Forward

My year placement at Magnite has had a massive impact on my professional development and my employability. At the start of this placement, I wasn’t sure what area of software development I wanted to go into. I felt the most confident in Python and maybe web development. When I was put into the SpringServe Reporting Team, which really only used Java to develop their software, I was surprised since I had very little experience with Java. This decision was probably based on my experience on my last placement where I built an attendance system in C#. But over the year, my skills and confidence in Java has improved substantially to the point where it is by far my favourite language.

It has also improved my professional development in other ways outside of programming. I now have experience in how software engineering teams work at an industrial level and how these projects are planned and implemented. The experience working in a team will be invaluable when looking for graduate roles.

Key Contributions

I set up a shared cloud file between all the computers in the office and each board members home computers, where any letters, bank statements and other documents could be saved to.

I created an attendance system using C# that is used by their childcare facility. Each day the attendance is logged into the system which is saved to an SQL database. The program allows a range of reports to be created from this data which are Tailor-made for their needs (invoices, reports sent to governing bodies, etc.).

My other roles included setting up their new computers and printers, creating tutorials on the new systems and security concerns, and helping board members with their own computer issues.
