NextGen Voting

A blockchain-based voting system I built in Django to test how encryption and smart contracts could be used to make online elections more secure and transparent.

View Code How it works

Overview

I wanted to see how a voting system could work if every vote was both encrypted and recorded on a public blockchain. The whole project runs in Django, with an async manager that handles writes to the Polygon network. Ballots are encrypted with Fernet before they ever touch the database or chain.

The system supports a few counting methods (like ranked choice) and is built around a clean structure that keeps the backend easy to extend.

How it works

  1. Users fill out a ballot form in Django.
  2. The vote data is encrypted before it’s saved.
  3. An async Blockchain Manager sends the encrypted hash to Polygon.
  4. Once confirmed, results can be tallied using different algorithms.
  5. Anyone can check the chain to verify transactions.

Tech stack

  • Python / Django
  • Polygon (EVM)
  • Fernet Encryption
  • Celery (Async Jobs)
  • SQLite / PostgreSQL

Current limitations

  • Prototype-only – no authentication or voter ID yet.
  • Polygon gas fees can cause small delays.
  • Frontend is basic; it’s mostly backend logic right now.

NextGen Voting 2.0

I’m planning a full rebuild that separates the backend and frontend completely. The backend will become a REST API built with Django REST Framework, and the frontend will be written in React.

  • Full React interface with live updates and better UX.
  • Django REST API for ballot submission, results, and auditing.
  • JWT-based authentication for voter sessions.
  • More modular blockchain layer to test different networks.
