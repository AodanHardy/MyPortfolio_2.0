Restaurant Management System

An idea I had for my final year project before deciding to go with something else. It’s a full restaurant system for handling orders, tables, and payments — all driven by a Java backend.

Overview

The goal was to build a lightweight system where staff could take orders and automatically route them to the correct place — either the bar or the kitchen. The backend is written in Java with a clean DAO layer, and data stored in PostgreSQL.

I still plan to finish it as a small side project — turning it into a public API and building a mobile app to connect to it. The app will let staff log in, select a table, place an order, and instantly send drinks to the bar or food to the kitchen.

Next steps

  • Finish REST API layer for orders, tables, and payments.
  • Develop a companion mobile app for staff devices.
  • Use WebSockets or MQTT to push new orders live to the bar or kitchen displays.
  • Add a lightweight admin panel for reports and menu updates.
